# 设计文档索引

本文档是 `doc/` 目录的统一入口，面向实现代理、设计代理和项目维护者，用于快速理解当前项目上下文、定位应读文档，并按正确顺序建立设计认知。

## 当前项目状态

- 项目阶段：前期设计阶段
- 当前目标：先固化项目母设计稿，再逐步拆分为可执行子设计文档
- 当前重点：验证 AI NPC 的循环思考、关系演化和单局叙事沙盒能力

## 推荐阅读顺序

实现代理进入项目后，默认按以下顺序阅读：

1. 先读 `00-master-design.md`
2. 再根据当前任务选择相应子文档
3. 如果任务涉及尚未拆分的主题，回到母设计稿对应章节作为当前规范

## 代理使用约定

### 实现代理

- 默认入口是 `README.md`
- 开工前至少阅读 `00-master-design.md`
- 如果某项子文档已存在，优先以子文档为直接执行规范
- 如果子文档不存在，以母设计稿对应章节为准，不要自行发明新的系统边界

### 设计代理

- 新增文档前先更新本索引
- 新文档必须明确自己的职责边界，避免与母稿或其他子稿重叠
- 新文档写完后，应在本索引中补充“用途”和“适用场景”

## 文档状态标记约定

目录中的文档默认按以下语义理解：

- `00-`：母设计稿或总入口
- `10-60`：主题化可执行设计文档
- `90-`：开放问题、风险项或待收敛主题

如果某个主题尚未拆出独立文档，则说明它仍由 `00-master-design.md` 承担。
