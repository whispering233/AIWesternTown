目标
先产出规范、布局说明和样式示例，再指导工程实现。
核心原则

保留纸面感和西部卷宗气质，通过留白、边框和分隔线建立秩序，而不是通过厚重卡片堆砌信息。

Play Surface

右栏承接系统上下文、目标、风险和 debug，不反客为主，不变成第二主栏。

Placement

进入主栏的条件

  • 它必须直接服务当前回合。
  • 它必须支持立刻决策或立刻反馈。
  • 如果更适合回看、检索或归档，就不要进主栏。
Overflow

主栏防过载规则

  • 首屏只允许一个 hero。
  • 首屏最多两个较重的信息块。
  • 完整历史流、索引和长文案优先拆到左栏或详情页。

Pattern Grammar

通用结构语法

页面家族会扩展，但 panel 语法、tabs 语法和行动语法应尽量稳定，这样设计和实现都更容易复用。

Panel Grammar

Eyebrow → Title → Description

每个 panel 先明确它只讲一件事，再通过分隔线和留白组织内容，不依赖厚重卡片背景。

Tabs Grammar

状态 / 事件流 / 日志

左栏默认三标签页，实时信息、沉淀信息和索引信息不能混写成同一种列表。

Action Grammar

3-5 个主动作足够

主要行动入口控制在 3-5 个，高频动作优先露出，其他动作通过分组或详情页承接。

Page Family

页面骨架示例

后续页面应基于同一套语言扩展，而不是重新发明各自的布局。

Scene Detail

场景深读页

世界栏 场景总览 / 分区 / 对象 风险 / 邻接信息
  • 世界栏保持不变
  • 主栏切换为场景分区、对象、在场角色
  • 右栏可放相关地点或风险提示
Character Sheet

角色页

日志 / 相关事件 身份卡 / 事实 / 关系 地点 / 风险
  • 左栏继续承载日志与事件流
  • 主栏 hero 改成角色身份卡
  • 主体为关系、事实、传闻、动态
Consequence View

后果页

事件上下文 结果 / 影响 / 新线索 目标 / 系统说明
  • 主栏以关键结果为中心
  • 保留影响扩散与新线索
  • 仍需保留下一步行动出口
Journal Timeline

日志页

过滤 / 索引 时间线 / ledger 统计 / 标签
  • 主栏转为 ledger 式纵向阅读
  • 左栏做过滤与索引
  • 适合沉淀线索与调查记录
Settlement Overview

据点总览页

地点日志 / 传闻 据点摘要 / 区域 / 可去地点 风险 / 旅行提示
  • 适合世界扩展到多个区域后使用
  • 主栏不做百科，而是服务下一步去向选择
  • 左栏继续承载沉淀式地点信息
Investigation Board

调查页

原始事件流 线索组 / 关系 / 假设 规则 / 目标
  • 适合中后期复杂线索组织
  • 主栏围绕问题和推理，而不是重复时间线
  • 原始事件仍回到左栏保留上下文

Do / Don't

迭代时的判断标准

Do

先定栏位职责，再放内容

先问清楚信息该进左栏、主栏还是右栏，再决定具体模块长什么样。

Do

把完整历史流收进世界栏

主栏只看当前回合最重要的动作和结果。

Don’t

不要把所有信息都塞进主栏

如果主栏同时承载历史流、索引、长说明和动作，说明布局已经失控。

Step 01

先判定 page family

新增页面先归类，不要从组件开始画。

Step 02

先定三栏职责

先决定哪些信息属于世界栏、主栏和系统栏。

Step 03

再补 panel 与视觉

确认骨架后再细化标题、按钮、状态和细节样式。
